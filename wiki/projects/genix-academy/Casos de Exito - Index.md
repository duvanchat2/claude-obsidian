## Cómo Documentar un Nuevo Caso

Al completar una implementación para un cliente, crear:

1. `wiki/projects/[nombre-cliente]/[Cliente] - Hub.md`
2. `wiki/projects/[nombre-cliente]/Estrategia [Cliente].md`
3. `wiki/projects/[nombre-cliente]/Resultados [Cliente].md`

Actualizar este índice con el nuevo caso.

### Checklist de documentación

- [ ] Contexto del cliente (sector, tamaño, desafío)
- [ ] Solución implementada (stack + flujo)
- [ ] Resultados con números reales
- [ ] Aprendizajes para replicar
- [ ] Testimonio del cliente (si autorizado)
- [ ] Plantilla replicable generada
